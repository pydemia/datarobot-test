report_predictions_data(features_df=None, predictions=None, association_ids=None, class_names=None, deployment_id=None, model_id=None)[source]¶

Report features and predictions to DataRobot MLOps for tracking and monitoring.

Parameters:
  • features_df (pandas dataframe) – Dataframe containing features to track and monitor. All the features in the dataframe are reported. Omit the features from the dataframe that do not need reporting.
  • predictions (list) – List of predictions. For a Regression deployments, this is 1D list containing prediction values. For a Classification deployment, this is a 2D list, in which the inner list is the list of probabilities for each class type Regression Predictions: e.g. [1, 2, 4, 3, 2] Binary Classification: e.g. [[0.2, 0.8], [0.3, 0.7]].

At least one of features or predictions should be specified.

Parameters:
  • association_ids (list) – an optional list of association IDs corresponding to each prediction. Used for accuracy calculations. Association IDs have to be unique for each prediction to report. Number of predictions should be equal to number of association_ids in the list
  • class_names (list) – names of predicted classes, e.g. [“class1”, “class2”, “class3”]. For a classification deployment, class names are defined during deployment creation. This field allows user to decide its own order for his/her convenience and is associated correctly by DataRobot MLOps. This argument is ignored for the Regression deployment.
  • deployment_id (str) – the deployment for these metrics
  • model_id (str) – the model for these metrics
Returns:

report status - True on success, False otherwise.
Return type:

bool
