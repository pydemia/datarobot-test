MLOps¶

public class MLOps¶

MLOps library provides metric reporting from external deployments to DataRobot MLOps.

reportPredictionsData¶

public boolean reportPredictionsData(String deploymentId, String modelId, Map<String, List<Object>> featureData, List<?> predictions, List<String> associationIds, List<String> classNames)¶

Report features and predictions with association ids and class names to DataRobot MLOps for tracking and monitoring.

Parameters:
  • deploymentId – the deployment being monitored
  • modelId – the model being monitored
  • featureData: – Map of feature name: list of values containing features to track and monitor. All the features in the map are reported. Omit the features from the map that do not need reporting.
  • predictions: – List of predictions. For a Regression deployments, this is 1D list containing prediction values. For a Classification deployment, this is a 2D list, in which the inner list is the list of probabilities for each class type Regression Predictions: e.g. [1, 2, 4, 3, 2] Binary Classification: e.g. [[0.2, 0.8], [0.3, 0.7]].
  • associationIds: – an optional list of association IDs corresponding to each prediction. Used for accuracy calculations. Association IDs have to be unique for each prediction to report. Number of predictions should be equal to number of associationIds in the list
  • classNames – names of predicted classes, e.g. [“class1”, “class2”, “class3”]. For a classification deployment, class names are defined during deployment creation. This field allows user to decide its own order for his/her convenience and is associated correctly by DataRobot MLOps. This argument is ignored for the Regression deployment. At least one of features or predictions should be specified.
Throws:
  • DRCommonException – when submission of reported features and predictions for the model fails
Returns:

report status - True on success, False otherwise.

setChannelConfig¶

public MLOps setChannelConfig(String channelConfig)¶

Configure the output channel to use given a single string configuration. The string can contain multiple configuration items separated by semicolon. Example: “spooler_type=FILESYSTEM;path=/tmp/spool_1;max_files=3;file_max_size=10000 First split the setting into key=value params then iterate over them setting the correct config key and value The config item names are the same as the environment variables used to configure the MLOps object. Note that config item names are case insensitive (e.g. SPOOLER_TYPE and spooler_type are valid)

Parameters:
  • channelConfig – a string containing the channel config
Throws:
  • DRApiException – for invalid channelConfig strings
  • DRVarNotFound – if a config variable provided is not known
  • DRInvalidValue – if the value provided to a config variable is not valid (e.g. provide integer to a variable expecting string).
Returns:

a reference to this object
